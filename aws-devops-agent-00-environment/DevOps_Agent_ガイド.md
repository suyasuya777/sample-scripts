# 🚀 AWS DevOps エージェント ガイド

> 🆕 印は **2026年6月（AWS Summit New York）で追加・強化された機能**。本ガイドでは新旧機能をまとめて整理している。

## 📌 1. サービス概要

**AWS DevOps エージェント**は、インシデントの調査・解決・予防を自律的に行う**フロンティアエージェント**。経験豊富な DevOps エンジニアのように動作し、24時間365日オンコール対応を実現する。さらにコードの**リリース前検証**まで担い、「書く」から「安全に出す」までをカバーする。

**主な効果指標**

| 指標 | 改善効果 |
|---|---|
| MTTR（平均解決時間） | 最大 75% 削減（数時間 → 数分） |
| 調査時間 | 最大 80% 短縮 |
| 根本原因特定精度 | 94% 達成 |

## 📌 2. 主要機能（4つの柱）

| 柱 | 概要 | ステータス |
|---|---|---|
| ① 自律型インシデント対応 | アラート受信と同時に自動調査・緩和 | GA |
| ② プロアクティブインシデント防止 | 過去パターンから予防的レコメンデーション | GA |
| ③ オンデマンド DevOps タスク | 自然言語チャットで調査・分析 | GA |
| ④ リリース管理 | コード変更を本番投入前に検証 | 🆕 プレビュー |

### 🔹 ① 自律型インシデント対応（常時オン）

- アラートまたはサポートチケット受信と同時に**自動調査開始**
- **詳細な緩和計画**（解決ステップ・検証・ロールバック）を生成
- Slack / ServiceNow 等のチャネルへ観察結果・緩和ステップを**自動ルーティング**
- AWS サポートケースをエージェントから直接作成
- 🆕 **インシデントスキップ**：顧客定義ルールに基づき、対応不要なインシデントを自動でスキップ

### 🔹 ② プロアクティブインシデント防止

- 過去インシデントのパターンを分析し、**予防的レコメンデーション**を生成
- 4領域を対象：オブザーバビリティ / インフラ最適化 / デプロイパイプライン / コード品質
- チームのフィードバックを取り込み、継続的に精度向上

### 🔹 ③ オンデマンド DevOps タスク（チャット）

- 自然言語でインフラクエリ・システム状態分析・調査を実行
- 表示ページに基づく**コンテキスト対応レスポンス**
- ファイル添付・アーティファクト生成・会話履歴管理をサポート

### 🔹 ④ リリース管理（🆕 プレビュー）

コード変更を**本番投入前**に評価する新しい柱。エージェントが「安全にリリースする」ところまで支援する。

| 機能 | 内容 |
|---|---|
| **リリースレディネスレビュー** | AWS管理の隔離環境でコードを実行し、ビルド・起動・基本的な機能チェック（軽量なユーザージャーニーテスト）を検証してからパイプラインへ通す |
| **自律リリーステスト** | 変更固有のテスト計画を生成し、本番相当の環境で実行 |

- 変更ごとに、エージェントへ与えた**自然言語の組織標準**に照らして検証
- 結果の確認先：**コンソール** / GitHub・GitLab の **PR コメント** / **Kiro・Claude Code プラグイン経由で IDE から**
- チャットから直接起動も可能（例：`Run a release test on my application deployed at [URL]`）
- **前提**：エージェントスペースに GitHub または GitLab リポジトリを最低1つ接続
- プレビュー期間中は追加料金なし（US East バージニア北部で提供）

## 📌 3. アーキテクチャと主要概念

### 🔹 デュアルコンソール構成

| コンソール | 利用者 | 主な用途 |
|---|---|---|
| **AWS マネジメントコンソール** | 管理者 | エージェントスペース作成・統合設定・IAM管理 |
| **DevOps エージェントウェブアプリ** | 運用チーム | インシデント対応・調査・防止レコメンデーション確認 |

### 🔹 エージェントスペース

**論理コンテナ**。エージェントがアクセスできるAWSアカウント・サードパーティーツール・ユーザー権限を定義する。

- **AWS アカウント分離**：専用 IAM ロールで範囲を制限
- **ユーザーアクセス分離**：チーム・環境ごとに権限を設定
- **データ分離**：調査データ・チャット履歴はスペース間で完全分離
- 複数スペースの使い分け例：チーム分離 / 本番・非本番 / コンプライアンス要件

### 🔹 トポロジ

AWSリソースとその関係を**自動検出・視覚化**するグラフ。調査・予防の基盤となる。

**検出方法**

| 方法 | 対象 |
|---|---|
| CloudFormation スタック | IaC（CDK含む）でデプロイされたリソース |
| AWS Resource Explorer | CloudFormation 外のタグ付きリソース |

**ビューの種類：** 学習済み / システム / コンテナ / コンポーネント / すべてのリソース

### 🔹 カスタムエージェント（🆕 スケジュール実行）

エージェントスペース内に、**定期実行（cadence）するカスタムエージェント**を作成・スケジュールできる。常時待機のインシデント対応とは別に、定常的な点検・レポートを自動化する用途。

- 例：スロークエリやチューニング対象パラメータを毎日チェックする **DB ヘルスレポート**
- 例：過去24時間のログをレビューし**異常を検出**するエージェント

### 🔹 タスク品質の追跡（🆕）

- **人手ラベリング**：エージェントのタスク結果を人がラベル付けし品質を評価
- **カスタムダッシュボード**：タスク品質を可視化・追跡

## 📌 4. ナレッジ（エージェントの知識を拡張する3要素）

エージェントの **Knowledge（ナレッジ）** は **Instructions / Skills / Memories** の3タブで構成される。それぞれ注入タイミングと用途が異なる。

| 要素 | 注入タイミング | 形式 | 役割 |
|---|---|---|---|
| **Instructions（AGENTS.md）** | 常時（無条件） | Markdown のみ | 全エージェントに常時適用する一般指示 |
| **Skills（SKILL.md）** | オンデマンド（説明マッチング） | Markdown / ZIP | 関連時にロードされる専門能力 |
| **Memories** 🆕 | 継続参照 | — | 過去の知見を蓄積し再利用 |

### 🔹 Instructions（エージェントの指示 / AGENTS.md）

**すべてのセッションに無条件で適用**される常時オンのガイダンス。スキルと異なりエージェントの判断でスキップされない。

- **適切な用途**：運用ポリシー / セキュリティガイドライン / レスポンスフォーマット標準
- **推奨サイズ**：120行以内（上限 25KB）
- リソースファイルは非サポート（Markdown 本文のみ）

### 🔹 Skills（スキル）

エージェントの機能を拡張する**モジュラー命令セット**。Markdown 形式の `SKILL.md` を中心に構成される。関連するタスクのときだけ自動ロードされる。

**スキルの構造**

```
my-skill/
├─ SKILL.md        # 必須：メイン指示（フロントマター + Markdown）
├─ references/     # 任意：参考資料
└─ assets/         # 任意：図・データファイル
```

**フロントマター（必須フィールド）**

```yaml
---
name: rds-performance-investigation
description: RDSパフォーマンス問題の調査手順。DBレイテンシー・接続エラー・
  クエリタイムアウトを調査するときに使用。
---
```

> ⚠️ `description` はエージェントがスキルをロードするかどうかの判断基準。具体的なシナリオ・サービス・症状を記述することが重要。

**エージェントタイプのターゲティング**

| エージェントタイプ | 用途 |
|---|---|
| 汎用（Generic） | すべてのエージェントタイプ（デフォルト） |
| オンデマンド | 会話型クエリ・チャット |
| インシデントトリアージ | 初期フィルタリング・重要度分類 |
| インシデント RCA | 根本原因分析 |
| インシデント緩和 | 短期修復・長期修正 |
| 評価 | パフォーマンススコアリング・ポリシーチェック |

**学習したスキル（自動生成）**

| スキル | 内容 | 更新タイミング |
|---|---|---|
| **エージェントスペース理解** | リソースマップ・アーキテクチャ図・リクエストパス | 統合追加/変更/削除時 |
| **ツールのベストプラクティス** | 調査ツールの効果的な使用パターン・障害モード | 30回の調査ごと |

**スキルの作成・管理方法**

- **UI作成**：名前・説明・手順を入力（SKILL.md を自動生成）
- **ZIP アップロード**：参考資料・アセット込みで最大 6MB
- 🆕 **Git管理スキル**：スキルを Git で管理し、バージョン管理・レビューを既存の開発フローに統合
- ※ スクリプト実行は現時点で未サポート（将来対応予定）

### 🔹 Memories（メモリ）🆕

過去のインシデント対応・調査から得た知見を蓄積し、エージェントが**継続的に参照**できるようにする3つ目のナレッジ要素。Instructions（常時適用）/ Skills（オンデマンド）とは別に、経験の積み上げを担う。

## 📌 6. 連携・アクセス方法

### 🔹 オブザーバビリティ（テレメトリソース）

| ツール | 統合タイプ |
|---|---|
| Amazon CloudWatch | 組み込み双方向 |
| Dynatrace | 組み込み双方向 |
| Datadog | 組み込み双方向（MCP経由） |
| New Relic | 組み込み一方向 |
| Splunk | 組み込み一方向 |
| Grafana | 組み込み一方向 |

### 🔹 CI/CD パイプライン

- **GitHub**（github.com / GitHub Enterprise Server）
- **GitLab**（gitlab.com / セルフホスト型）

### 🔹 チケット・チャット

- PagerDuty / ServiceNow / Slack

### 🔹 MCP（Model Context Protocol）サーバー — ツールを「接続する」

カスタムツールをエージェントに接続し、調査機能を拡張する（**エージェント ← ツール**の方向）。

- 要件：Streamable HTTP トランスポート + OAuth 2.0 / API キー / SigV4 認証
- セキュリティ：必要なツールのみ許可リスト登録を推奨

### 🔹 A2A / ヘッドレスアクセス — 外部ツールから「呼び出す」🆕

既存ツールから DevOps Agent を起動する、MCP とは**逆方向**の連携（**外部ツール → エージェント**）。

- **A2A（Agent-to-Agent）/ MCP プロトコル**経由で、Kiro・Claude・その他コーディングアシスタントから起動
- **Kiro power for AWS DevOps Agent**：IDE を離れずに本番健全性チェック・問題調査が可能
- **独自サブエージェントの持ち込み**：Amazon Bedrock やサードパーティフレームワークで構築したサブエージェントを A2A で接続して機能拡張

### 🔹 その他の連携

- **Azure**（Azure リソース / Azure DevOps）
- **Amazon EventBridge**（エージェントイベントのルーティング）
- **AWS PrivateLink**（プライベート接続）
- **Amazon CloudWatch メトリクス・ログ**（モニタリング）

## 📌 7. セキュリティ

### 🔹 マルチレイヤー保護

| 保護レイヤー | 内容 |
|---|---|
| **IAM ロール** | 最小権限原則。エージェントは読み取り系のみ（チケット作成を除き変更不可） |
| **エージェントスペース境界** | 設定範囲外のリソースへのアクセス不可 |
| **AI 安全保護（ASL-3）** | プロンプトインジェクション検出・防止の分類器を内蔵 |
| **イミュータブル監査証跡** | エージェントジャーナルは変更不可。全推論ステップを記録 |
| **AWS CloudTrail 統合** | 全 API コールを自動キャプチャ |
| **データ暗号化** | 保管時・転送中ともに暗号化。カスタマーマネージドキー（CMK）対応 |

### 🔹 責任共有モデル

- **AWS 責任**：コアサービスのセキュリティ・ASL-3保護・監査証跡
- **お客様責任**：IAM ポリシー設定・MCP サーバーの検証・データソースの信頼性確保

### 🔹 認証方式

| 方式 | 用途 |
|---|---|
| IAM Identity Center | ユーザーアクセス管理（SAML/OIDC フェデレーション対応） |
| 外部 IdP（Okta / Microsoft Entra ID） | OIDC 互換 IdP の直接接続 |
| IAM 認証リンク | 管理者向け一時アクセス（セッション10分） |
