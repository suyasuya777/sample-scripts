# 📘 AWS DevOps Agent 初期設定手順

AWS マネージメントコンソールから AWS DevOps Agent を使用開始するまでの手順を説明します。

---

## 🧰 前提条件

- AWS マネージメントコンソールへのアクセス権限があること
- 対象リージョン: **東京（ap-northeast-1）**

---

## 🧰 セットアップ手順

### ⚙️ Step 1 　AWS DevOps Agent を選択

AWS マネージメントコンソールにサインインし、サービス一覧から **AWS DevOps Agent** を選択します。

---

### ⚙️ Step 2 　リージョン確認 ＆ セットアップ開始

1. コンソール右上のリージョン表示が **東京（ap-northeast-1）** であることを確認
2. **「セットアップを開始」** をクリック

> ⚠️ リージョンが異なる場合は、東京リージョンに切り替えてから進めてください。

---

### ⚙️ Step 3 　エージェントスペースを作成

1. 新規作成するエージェントスペース名を入力
2. **「保存」** をクリック

保存が完了すると、以下のロールが **自動作成** されます。

| ロール | 説明 |
|---|---|
| エージェントスペースロール | エージェントが AWS リソースへアクセスするためのロール |
| ウェブアプリロール | ウェブアプリケーション用のアクセスロール |

---

### ⚙️ Step 4 　オペレーターアクセスを開く

作成されたエージェントスペースの画面で **「オペレーターアクセス」** をクリックします。

---

### ⚙️ Step 5 　インシデントレスポンスから調査を開始

**「インシデントレスポンス」** セクションを選択し、調査を始めます。

---

### ⚙️ Step 6 　調査を開始

**「調査を開始」** をクリックします。

---

### ⚙️ Step 7 　調査の詳細を入力

以下の項目を入力後、**「調査を開始」** をクリックします。

| 項目 | 説明 |
|---|---|
| 調査の詳細 | 調査の目的・背景など概要を記入 |
| 調査の出発点 | 調査を開始するきっかけとなった事象や情報を記入 |

---

## 🧰 クロスアカウント対応：セカンダリリソースの追加

別アカウントのリソースも調査対象に含めたい場合は、エージェントスペースに **セカンダリリソース** として別アカウントを追加します。エージェント側アカウントのロールが、追加先アカウントに作成したロールを引き受けることで、クロスアカウントでの調査が可能になります。

### ⚙️ 用語と前提

本手順では 2 つのアカウントを行き来します。それぞれ次のとおり呼び分けます。

| 本手順での呼称 | 役割 | 正式名称 |
|---|---|---|
| 信頼される側のアカウント（左側の列に記載） | エージェント側。エージェントスペースがあり、調査を実行する（＝アクセスする側） | Trusted account |
| 信頼する側のアカウント（右側の列に記載） | 追加先。調査対象のリソースを持ち、エージェントにロールの引き受けを許可する（＝アクセスされる側） | Trusting account |

> 💡 2 つのアカウントの画面を同時に操作します。**信頼される側は通常ウィンドウ**、**信頼する側はシークレットウィンドウ（Ctrl + Shift + N）** でログインしておくと、両方のログインを保ったまま値をコピー＆ペーストできます。

---

### ⚙️ Step A 　【事前準備】両アカウントへログイン

| 信頼される側（通常ウィンドウ） | 信頼する側（シークレットウィンドウ） |
|---|---|
| 通常のウィンドウでログインします。 | シークレットウィンドウ（**Ctrl + Shift + N**）でログインします。 |

---

### ⚙️ Step B 　【信頼される側】セカンダリリソースの追加を開始

| 信頼される側（通常ウィンドウ） | 信頼する側（シークレットウィンドウ） |
|---|---|
| 1. 対象の **エージェントスペース** を選択します。 |  |
| 2. **エージェントの機能 ＞ セカンダリリソース** で **「追加」** を選択します。 |  |
| 3. **「ソースアカウントを追加」** で **「AWS」** を選択します。 |  |
| 4. 追加先で作成するロールに付ける **ロール名** を入力します。 |  |

> このウィザードには、後続で使う **カスタム信頼ポリシー（ステップ 2）** と **インラインポリシーの JSON（ステップ 6）** が表示されます。**この画面は閉じずに開いたまま**にしておきます。

---

### ⚙️ Step C 　【信頼する側】IAM ロールとポリシーを作成

> 以降はシークレットウィンドウの **信頼する側のアカウント** で操作します。

| 信頼される側（通常ウィンドウ） | 信頼する側（シークレットウィンドウ） |
|---|---|
|  | 1. IAM で **ロールを作成** します。 |
|  | 2. 信頼されたエンティティで **「カスタム信頼ポリシー」** を選択し、**「次へ」** をクリックします。 |
|  | 3. Step B のウィザードの **ステップ 2** に表示された **カスタム信頼ポリシー** をコピーして貼り付け、**「次へ」** をクリックします。 |
|  | 4. 許可ポリシーで **`AIDevOpsAgentAccessPolicy`** を選択し、**「次へ」** をクリックします。 |
|  | 5. **ロール名・説明（任意）・タグ（任意）** を入力し、**「ロールを作成」** をクリックします。 |
|  | 6. 作成したロールを開きます。 |
|  | 7. **「許可」** タブ ＞ **「許可を追加」 ＞ 「インラインポリシーを追加」** をクリックします。 |
|  | 8. **「JSON」** を選択し、Step B のウィザードの **ステップ 6** に表示された JSON をコピーして貼り付けます。 |
|  | 9. JSON 内の **`<YOUR_ACCOUNT_ID>`** を、**信頼する側（操作中のこのアカウント）のアカウント ID** に書き換えて **「次へ」** をクリックします。 |
|  | 10. **ポリシー名** を入力し、**「ポリシーの作成」** をクリックします。 |

---

### ⚙️ Step D 　【信頼される側】アカウント ID を入力して有効化

> 通常ウィンドウの **信頼される側のアカウント** に戻って操作します。

| 信頼される側（通常ウィンドウ） | 信頼する側（シークレットウィンドウ） |
|---|---|
| 1. Step B のウィザードで **「次へ」** を選択します。 |  |
| 2. **アカウント ID** に **信頼する側のアカウントのアカウント ID** を入力し、**「追加」** をクリックします。 |  |
| 3. **セカンダリリソースのステータスが「有効」** になっていることを確認します。 |  |
