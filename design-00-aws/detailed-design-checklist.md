# インフラ詳細設計チェックリスト（AWS版）

> 基本設計チェックリストが「要件を満たす実現方式・構成方針が決定されているか」を確認するのに対し、  
> 本チェックリストは **「実装可能なレベルまでパラメータ・設定値が確定しているか」** を確認します。

---

## 1. 目的・スコープ（詳細）

### 1.2 環境構成一覧

- [ ] dev/stg/prodのAWSアカウントIDを確定した
- [ ] 環境ごとのアクセス制御方針（Assume Role等）を明記した

### 1.3 命名規則

- [ ] 全リソース種別の命名パターンを定義した
- [ ] `{env}` `{system}` 等のプレースホルダーを実際の値に置換可能な形にした
- [ ] Terraformリソース名の命名規則（スネークケース等）を定義した

### 1.4 タグ付け規則

- [ ] 必須タグキー（Project/Environment/ManagedBy/Owner/CostCenter等）を定義した
- [ ] CostCenterコード等、組織固有の値を確定した

### 1.5 / 1.6 Terraform構成

- [ ] ディレクトリ構成（environments/modules/shared）を確定した
- [ ] Backend用S3バケット名・DynamoDBロックテーブル名を確定した
- [ ] State用バケットのバージョニング・暗号化設定を確認した

## 2. システム概要（詳細）

### 2.1 アーキテクチャ構成図・通信表

- [ ] アーキテクチャ図を作成・添付した
- [ ] コンポーネント間通信の送信元/宛先/プロトコル/ポートを一覧化した
- [ ] 通信表とセキュリティグループ設定（3.6）の整合性を確認した

### 2.2 非機能要件（数値詳細）

- [ ] 各非機能要件項目に「測定方法」「測定period」を明記した
- [ ] レスポンスタイムのp50/p95/p99を個別に定義した

### 2.3 環境別スペック差分サマリ

- [ ] dev/stg/prodの差分項目を一覧化した
- [ ] 差分項目が後続章（4〜10）の各表と矛盾していないか確認した

---

## 3. ネットワーク設計（詳細）

### 3.1 VPC詳細設定

- [ ] 環境ごとのCIDRブロックを確定した（dev/stg/prod間で重複なし）
- [ ] DNS解決・DNSホスト名の設定値を確認した
- [ ] 利用AZを確定した

### 3.2 サブネット詳細設定

- [ ] 各サブネットのCIDR・AZ・ルートテーブル対応を一覧化した
- [ ] CIDR範囲がVPC全体のCIDR内に収まっているか確認した
- [ ] dev/stg環境のAZ数削減方針を明記した（コスト削減対象の場合）

### 3.3 ルートテーブル詳細

- [ ] Public/Private/DBサブネットごとのルートテーブルを定義した
- [ ] DBサブネットに外部経路（0.0.0.0/0）が含まれていないことを確認した
- [ ] PrivateサブネットのNAT Gateway向けルートがAZごとに正しく設定されているか確認した

### 3.4 Internet Gateway / NAT Gateway

- [ ] prod環境のNAT Gateway数（AZ冗長）を確定した
- [ ] dev/stg環境のNAT Gateway削減方針（単一AZ）を確認した
- [ ] Elastic IPの割当数を確認した

### 3.5 VPC Endpoint詳細設定

- [ ] 必要なVPC Endpoint（ECR/S3/Logs/SSM/Secrets Manager等）を全て一覧化した
- [ ] Interface型エンドポイントの配置サブネット・SGを確定した
- [ ] Private DNS有効化設定を確認した
- [ ] Gateway型（S3）のルートテーブル関連付けを確認した

### 3.6 セキュリティグループ詳細

- [ ] ALB/ECS/RDS/Cache/VPCEごとにSGを分離した
- [ ] 各SGのInbound/Outboundルールがソース/宛先をSG ID参照で記述しているか確認した
- [ ] 通信表（2.1）とSGルールの対応に過不足がないか確認した
- [ ] 不要なOutbound全許可（0.0.0.0/0）が残っていないか確認した

---

## 4. コンピューティング設計（詳細）

### 4.1 / 4.2 ECSクラスター・タスク定義

- [ ] クラスター名・Capacity Provider・Container Insights設定を確認した
- [ ] 環境別のCPU/Memoryサイズを確定した
- [ ] コンテナ定義（イメージ/ポート/ログ設定/ヘルスチェック）を確定した
- [ ] ヘルスチェックパスがアプリケーション側の実装と一致しているか確認した

### 4.2 環境変数・シークレット

- [ ] 環境変数とSecrets Manager参照の振り分けを確定した
- [ ] シークレットのARN形式・キー名を確定した
- [ ] アプリ側でハードコードされた認証情報が残っていないか確認した

### 4.3 ECSサービス設定

- [ ] 環境ごとのDesired Count・配置サブネットを確定した
- [ ] デプロイコントローラ（Blue/Green）の設定を確認した
- [ ] ヘルスチェック猶予期間がアプリ起動時間に対して十分か確認した

### 4.4 Auto Scaling詳細設定

- [ ] Min/Max Capacityを環境ごとに確定した
- [ ] スケーリングメトリクス・ターゲット値・クールダウン秒数を確定した
- [ ] prod環境でCPU以外の追加ポリシー（Memory等）の要否を検討した

### 4.5 タスクロール・実行ロール詳細

- [ ] タスク実行ロールとタスクロールのアタッチポリシーを確定した
- [ ] インラインポリシーのResource ARNが過度に広くなっていないか確認した
- [ ] 権限境界（Permission Boundary）の要否を判断した

### 4.6 Lambda設定（使用する場合）

- [ ] ランタイム・メモリ・タイムアウトを確定した
- [ ] VPC配置の要否とサブネットを確定した
- [ ] 同時実行数上限・DLQ設定を確認した

## 5. データ層設計（詳細）

### 5.1 RDS / Auroraクラスター詳細

- [ ] エンジンバージョンを確定した（dev/stg/prod統一）
- [ ] 環境ごとのインスタンスクラス・インスタンス数を確定した
- [ ] マルチAZ設定がprodで有効になっているか確認した
- [ ] パラメータグループ・サブネットグループ名を確定した

### 5.2 RDSパラメータグループ詳細値

- [ ] `max_connections`等の主要パラメータ値を確定した
- [ ] スロークエリログ閾値（`log_min_duration_statement`）を確定した
- [ ] `rds.force_ssl`等のセキュリティ関連パラメータを確認した

### 5.3 バックアップ・メンテナンス設定

- [ ] 環境ごとのバックアップ保持期間を確定した
- [ ] バックアップウィンドウ・メンテナンスウィンドウが業務時間外か確認した
- [ ] prod環境で削除保護・最終スナップショット取得が有効か確認した
- [ ] Performance Insightsの有効/無効・保持期間を確認した

### 5.4 認証情報管理（Secrets Manager）

- [ ] シークレット名・自動生成パスワードのポリシー（長さ/除外文字）を確定した
- [ ] prod環境のローテーション設定（周期・Lambda）を確認した

### 5.5 S3バケット詳細

- [ ] バケットごとの用途・バージョニング・暗号化設定を一覧化した
- [ ] パブリックアクセスブロックが全バケットで有効か確認した
- [ ] ライフサイクルルール（Glacier移行/削除/中断アップロード削除）を確定した
- [ ] バケットポリシーのPrincipal/Actionが最小権限になっているか確認した

### 5.6 ElastiCache詳細設定（使用する場合）

- [ ] エンジンバージョン・ノードタイプ・ノード数を確定した
- [ ] サブネットグループ・暗号化（保存時/転送時）設定を確認した
- [ ] prod環境で自動フェイルオーバーが有効か確認した

## 6. セキュリティ設計（詳細）

### 6.1 / 6.2 IAMロール・ポリシー詳細

- [ ] 全IAMロールの信頼関係（Principal）を確認した
- [ ] 各ロールのアタッチポリシーが最小権限になっているか確認した
- [ ] インラインポリシーのResource ARNにワイルドカードが不要に使われていないか確認した
- [ ] CI/CD用ロール（Terraform実行ロール等）の権限範囲を環境ごとに分離した

### 6.3 KMSキー設計

- [ ] RDS/S3/Secrets Manager等、暗号化対象ごとに使用キーを確定した
- [ ] prodでカスタムKMSキーを使う場合のキーローテーション設定を確認した

### 6.4 ACM証明書設定

- [ ] ドメイン名・検証方法（DNS検証）を確定した
- [ ] ALB用とCloudFront用でリージョンが異なる点を確認した（CloudFront用はus-east-1）
- [ ] 証明書の自動更新設定を確認した

### 6.5 WAF詳細設定

- [ ] WebACL名・スコープ（REGIONAL/CLOUDFRONT）を確定した
- [ ] マネージドルールグループの優先度を確定した
- [ ] レートベースルールの閾値（リクエスト数/期間）を確定した
- [ ] WAFログの送信先・保持期間を確認した

### 6.6 セキュリティサービス詳細設定

- [ ] GuardDuty/CloudTrail/Config/SecurityHub/Inspectorの有効化設定を確認した
- [ ] CloudTrailのマルチリージョン・ログファイル検証が有効か確認した
- [ ] AWS Configの評価ルール一覧を確定した
- [ ] SecurityHubで有効化する標準（CIS等）を確定した
