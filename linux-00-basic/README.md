# 🛠️ SRE向け BASH サンプル集

SREが日常的に使用するコマンドのサンプルスクリプト集です。  
各スクリプトはサンプルデータを自動生成するため、そのまま `bash` で実行できます。

```bash
# プロセス・リソース監視系
bash ps_top.sh
bash lsof.sh
bash strace.sh
bash vmstat_iostat.sh

# ネットワーク系
bash curl.sh
bash ss.sh
bash dig.sh
bash tcpdump.sh

# ツール系
bash grep.sh
bash sed.sh
bash awk.sh
bash jq.sh
bash find.sh
```

## ⚙️ プロセス・リソース監視系

### 📊 [ps_top.sh](ps_top.sh)

プロセスの一覧・状態確認とCPU/メモリ使用量を監視するコマンドです。  
`top` / `htop` はリアルタイムモニタリングにも使用します。

| No | タイトル | 概要 |
|---|---|---|
| 1 | 全プロセス一覧 | `ps aux` で全プロセスをユーザー・CPU・メモリ付きで一覧表示する |
| 2 | プロセス名で検索 | `ps aux \| grep` で特定プロセスを名前で絞り込む |
| 3 | CPU使用率上位 | `--sort=-%cpu` でCPU使用率の高い順に並べて上位5件を表示する |
| 4 | メモリ使用率上位 | `--sort=-%mem` でメモリ使用率の高い順に並べて上位5件を表示する |
| 5 | ユーザー別プロセス | `-u` オプションで特定ユーザーのプロセスのみ表示する |
| 6 | プロセスツリー | `axjf` で親子関係をツリー形式で表示する |
| 7 | PID指定の詳細表示 | `-p` でPIDを指定し、カスタムフォーマットで詳細情報を表示する |
| 8 | カスタムフォーマット | `-eo` でPID・プロセス名・CPU・メモリ・起動時刻を任意の形式で表示する |
| 9 | ゾンビプロセス確認 | STATが `Z` のプロセスをawkで検出する |
| 10 | ユーザー別集計 | awk でユーザーごとのプロセス数を集計してソート表示する |
| 11 | topバッチモード | `-b -n 1` でtopを非対話モードで1回だけ実行してスクリプトから利用する |
| 12 | top 特定PID監視 | `-p PID` で特定プロセスだけをtopで監視する |
| 13 | htop の使い方 | インタラクティブTUIツールとしての主な起動オプション・キー操作をまとめる |
| 14 | 高CPU自動検出 | 閾値超えのプロセスをawkで自動検出して警告メッセージを出力する |
| 15 | メモリ使用量MB表示 | RSSをMB換算してメモリ使用量上位5プロセスを整形表示する |

### 🔍 [lsof.sh](lsof.sh)

開いているファイルやネットワーク接続を確認するコマンドです。  
ポート競合の調査やディスクの削除済みファイル残存確認に有用です。

| No | タイトル | 概要 |
|---|---|---|
| 1 | 全オープンファイル数 | `lsof \| wc -l` でシステム全体のオープンファイル総数を確認する |
| 2 | PID指定 | `-p PID` で特定プロセスが開いているファイルを表示する |
| 3 | ユーザー指定 | `-u` で特定ユーザーが開いているファイルを表示する |
| 4 | 特定ファイルを開くプロセス | ファイルパスを引数に渡して、そのファイルを開いているプロセスを表示する |
| 5 | ディレクトリ配下を検索 | `+D` でディレクトリ配下のファイルを開いているプロセスを検索する |
| 6 | ネットワーク接続一覧 | `-i` でTCP/UDPの全接続を表示する |
| 7 | 特定ポートの確認 | `-i :PORT` でそのポートを使用しているプロセスを特定する |
| 8 | LISTENポート一覧 | `-i -sTCP:LISTEN` でLISTEN中のポートとプロセスを一覧表示する |
| 9 | ESTABLISHED接続 | `-i -sTCP:ESTABLISHED` で確立済みのTCP接続を表示する |
| 10 | 削除済みファイル検出 | `grep "(deleted)"` で削除済みだが開いたままのファイルを確認する |
| 11 | プロセス名でフィルタ | `-c` でプロセス名を指定してフィルタする |
| 12 | プロセス別FD数集計 | awk でプロセスごとのオープンファイル数を集計してソートする |
| 13 | ポート使用確認スクリプト | if文と組み合わせてポートの使用/未使用を自動判定する |
| 14 | プロセスの接続数確認 | 特定プロセスのESTABLISHED接続数をカウントして表示する |

### 🔬 [strace.sh](strace.sh)

プロセスが発行するシステムコールをトレースするコマンドです。  
アプリが読み込む設定ファイルの特定や、遅延・エラーの根本原因調査に使用します。

| No | タイトル | 概要 |
|---|---|---|
| 1 | 基本トレース | コマンドをstrace経由で実行し全システムコールを表示する |
| 2 | 統計サマリー（`-c`） | `-c` でシステムコールごとの呼び出し回数・時間をサマリー表示する |
| 3 | 特定syscallのみ（`-e`） | `-e trace=openat` など特定のシステムコールだけをフィルタして表示する |
| 4 | ファイルに保存（`-o`） | `-o` で結果をファイルに保存して後から解析する |
| 5 | タイムスタンプ付与（`-tt`） | `-tt` でマイクロ秒精度のタイムスタンプを付与する |
| 6 | 所要時間表示（`-T`） | `-T` で各システムコールの実行時間を表示して遅い処理を特定する |
| 7 | 実行中プロセスにアタッチ（`-p`） | `-p PID` で起動中のプロセスにアタッチしてリアルタイムトレースする |
| 8 | 子プロセスもトレース（`-f`） | `-f` でfork/execした子プロセスも含めてトレースする |
| 9 | 文字列長の変更（`-s`） | `-s 256` でデフォルト32文字から拡張してパラメータを完全表示する |
| 10 | ファイルアクセス系 | `-e trace=file` でファイル操作に関するsyscallのみ表示する |
| 11 | プロセス生成系 | `-e trace=process` でfork/exec等のプロセス操作syscallのみ表示する |
| 12 | 設定ファイルの特定 | `openat` をトレースしてアプリが読み込むファイルパスを特定する |
| 13 | 遅いsyscallの特定 | `-T` の出力をawkで集計して平均所要時間を算出する |
| 14 | エラーsyscallのみ表示 | `= -1` でフィルタしてエラーになったシステムコールだけを抽出する |

### 📈 [vmstat_iostat.sh](vmstat_iostat.sh)

仮想メモリ・CPU・ディスクI/O統計を表示するコマンドです。  
スワップ発生やI/O待機によるボトルネックの特定に使用します。

| No | タイトル | 概要 |
|---|---|---|
| 1 | vmstat 基本表示 | 現在のプロセス・メモリ・スワップ・I/O・CPU統計を一覧表示する |
| 2 | 定期出力（インターバル指定） | `vmstat 2 5` で2秒間隔で5回繰り返し出力して傾向を確認する |
| 3 | MB単位で表示（`-S M`） | `-S M` でメモリ関連の値をMB単位に変換して表示する |
| 4 | ディスク統計（`-d`） | `-d` でデバイスごとの読み書き回数・セクタ数を表示する |
| 5 | スラブキャッシュ統計（`-m`） | `-m` でカーネルのメモリアロケータ（slab）の使用状況を表示する |
| 6 | タイムスタンプ付き（`-t`） | `-t` で出力行に時刻を付与してログとして保存しやすくする |
| 7 | スワップ監視 | swpdフィールドをawkで抽出してスワップ使用の有無を自動判定する |
| 8 | iostat 基本表示 | CPUとデバイス別のI/O統計をまとめて表示する |
| 9 | デバイス別詳細（`-x`） | `-x` でr/s・w/s・await・%utilなどの詳細I/O統計を表示する |
| 10 | MB単位で表示（`-m`） | `-m` でI/O量をMB/s単位に変換して表示する |
| 11 | CPUのみ表示（`-c`） | `-c` でCPU統計（us/sy/id/wa/st）のみを繰り返し表示する |
| 12 | 特定デバイス指定 | デバイス名を引数に渡して対象ディスクのみの統計を表示する |
| 13 | I/O待機率の監視 | vmstatのwaフィールドを抽出して閾値と比較し自動警告する |
| 14 | リソースサマリー | vmstat・iostatを組み合わせてCPU/メモリ/ディスクを1画面で表示する |

## 🌐 ネットワーク系

### 🌍 [curl.sh](curl.sh)

HTTPリクエストを送信するコマンドです。  
APIのヘルスチェック・テスト・レスポンスタイム計測などSREで最も頻繁に使用します。

| No | タイトル | 概要 |
|---|---|---|
| 1 | 基本GETリクエスト | URLを指定してGETリクエストを送信しレスポンスを表示する |
| 2 | ステータスコードのみ確認 | `-o /dev/null -w "%{http_code}"` でステータスコードだけを取得する |
| 3 | ヘルスチェック | ステータスコードをif文で判定して正常/異常を自動判定する |
| 4 | レスポンスタイム計測 | `-w` でDNS解決・TCP接続・TLS・最初のバイトまでの時間を詳細計測する |
| 5 | POST（JSONボディ） | `-X POST -H "Content-Type: application/json" -d` でJSONをPOSTする |
| 6 | カスタムヘッダー | `-H` でリクエストヘッダーを追加する（認証トークン・リクエストIDなど） |
| 7 | リダイレクト追跡（`-L`） | `-L` でリダイレクトを自動追跡し最終URLとリダイレクト回数を確認する |
| 8 | タイムアウト設定 | `--connect-timeout` / `--max-time` で接続・全体のタイムアウトを設定する |
| 9 | ファイルダウンロード | `-o` で指定ファイル名、`-O` でリモートファイル名でダウンロードする |
| 10 | レスポンスヘッダー表示 | `-I` でヘッダーのみ、`-v` で送受信の詳細を表示する |
| 11 | 基本認証 | `-u user:passwd` でBasic認証付きリクエストを送信する |
| 12 | プロキシ設定 | `-x` オプションまたは環境変数でHTTPプロキシ経由でリクエストする |
| 13 | SSL検証スキップ（`-k`） | `-k` でSSL証明書の検証をスキップする（開発・検証環境のみ） |
| 14 | 複数エンドポイント監視 | for文でエンドポイントを繰り返しチェックしステータスと応答時間を一覧表示する |
| 15 | jqと組み合わせてパース | curlのレスポンスをパイプでjqに渡して特定フィールドを抽出する |

### 🔌 [ss.sh](ss.sh)

ネットワーク接続・ポートの状態を確認するコマンドです。  
`netstat`（net-tools）は現代のLinuxでは非推奨のため、後継の `ss` に統一しています。

| No | タイトル | 概要 |
|---|---|---|
| 1 | 全ソケット一覧 | `ss` で全ソケット（TCP/UDP/Unix）を一覧表示する |
| 2 | LISTENポート一覧（TCP） | `ss -tlnp` でLISTEN中のTCPポートとプロセス名を一覧表示する |
| 3 | LISTENポート一覧（UDP） | `ss -ulnp` でLISTEN中のUDPポートとプロセス名を一覧表示する |
| 4 | ESTABLISHED接続 | `ss -tn state established` で確立済みのTCP接続を表示する |
| 5 | 特定ポートの接続確認 | `grep ":PORT"` で特定ポートの接続状態を絞り込む |
| 6 | 接続状態の集計 | awk でTCPの状態（LISTEN/ESTABLISHED/TIME_WAIT等）ごとの件数を集計する |
| 7 | 特定プロセスのソケット | grepでプロセス名を絞り込んでそのプロセスのソケットのみ表示する |
| 8 | バッファサイズ表示（`-m`） | `-m` で送受信バッファのサイズを表示する |
| 9 | Unixドメインソケット | `ss -xlnp` でUnixドメインソケットの一覧を表示する |
| 10 | TIME_WAIT確認 | `state time-wait` でTIME_WAIT状態の接続数を確認する |
| 11 | ポート開放確認スクリプト | if文でポートのLISTEN状態を自動判定して結果を出力する |
| 12 | 接続数の閾値監視 | ESTABLISHED接続数をカウントして閾値超えで警告を出力する |

### 🌐 [dig.sh](dig.sh)

DNS名前解決を行うコマンドです。  
`nslookup` はスクリプト利用では非推奨のため `dig` に統一しています。障害時のDNS疎通確認や、キャッシュ・伝播状況の調査に使用します。

| No | タイトル | 概要 |
|---|---|---|
| 1 | 基本検索 | `dig example.com A` でAレコード（IPアドレス）を検索する |
| 2 | 簡潔出力（`+short`） | `+short` でIPアドレスのみをシンプルに表示する |
| 3 | 各レコードタイプ検索 | MX・NS・TXT・CNAMEなど各レコードタイプを指定して検索する |
| 4 | 逆引きDNS | `-x IPアドレス` でPTRレコードを逆引きしてホスト名を取得する |
| 5 | 特定DNSサーバーへの問い合わせ | `@8.8.8.8` のようにDNSサーバーを指定して問い合わせる |
| 6 | TTLの確認 | `awk` でTTLフィールドを抽出してキャッシュ有効期間を確認する |
| 7 | DNSSEC確認（`+dnssec`） | `+dnssec` でDNSSEC署名レコード（RRSIG）を確認する |
| 8 | 応答時間の計測 | `+stats` でQuery timeを確認してDNSサーバーの応答速度を計測する |
| 9 | SOAレコード確認 | SOAレコードでゾーンの権威情報・シリアル番号を確認する |
| 10 | トレースモード（`+trace`） | `+trace` でルートDNSからの名前解決の経路を段階的に表示する |
| 11 | 複数ドメインの応答時間比較 | for文で複数ドメインのDNS解決時間とIPアドレスを一括表示する |
| 12 | DNSサーバー間の解決結果比較 | Google/Cloudflare/システムDNSの解決結果を並べて比較する |

### 📡 [tcpdump.sh](tcpdump.sh)

ネットワークパケットをキャプチャして解析するコマンドです。  
通信が届いているかの確認や、HTTPの内容・TCP接続異常の調査に使用します。  
※ root権限または CAP_NET_RAW 権限が必要です。

| No | タイトル | 概要 |
|---|---|---|
| 1 | 基本キャプチャ（`-i`） | インターフェースを指定してパケットをキャプチャする |
| 2 | インターフェース一覧 | `-D` で利用可能なネットワークインターフェースを一覧表示する |
| 3 | ホストフィルタ | `host`・`src host`・`dst host` で特定ホストの通信を絞り込む |
| 4 | ポートフィルタ | `port`・`portrange` で特定ポートの通信を絞り込む |
| 5 | プロトコルフィルタ | `tcp`・`udp`・`icmp` でプロトコルを指定してフィルタする |
| 6 | ファイル保存と読み込み | `-w` でpcap形式に保存し `-r` で読み込んで後から解析する |
| 7 | パケット内容表示（`-A` / `-X`） | `-A` でASCII、`-X` でHEX+ASCIIでパケットの内容を表示する |
| 8 | タイムスタンプ変更（`-tttt`） | `-tttt` で絶対時刻形式のタイムスタンプを付与する |
| 9 | 複合フィルタ | `and`・`or`・`not` を組み合わせて複雑な条件でフィルタする |
| 10 | キャプチャサイズ制限（`-s`） | `-s 0` で全データ、`-s 96` でヘッダーのみをキャプチャする |
| 11 | HTTPリクエストの確認 | `-A` とgrepを組み合わせてHTTPのURL・ヘッダー・ステータスを確認する |
| 12 | SYNパケットのみキャプチャ | TCPフラグでSYN/SYN-ACK/RSTのみをフィルタして接続状態を調査する |
| 13 | 障害調査フロー | キャプチャ→確認→RST検出→再送確認の標準的な障害調査手順を示す |
| 14 | DNS問い合わせの確認 | ポート53をキャプチャして問い合わせているドメイン名を確認する |

### 🔍 [grep.sh](grep.sh)

テキストファイルやログからパターンに一致する行を検索するコマンドです。  
サンプルデータ：アプリケーションログ (`/tmp/app.log`)

| No | タイトル | 概要 |
|---|---|---|
| 1 | 基本検索 | `grep "ERROR"` で ERRORを含む行を抽出する |
| 2 | 大文字小文字を無視（`-i`） | `-i` オプションで大文字小文字を区別せず検索する |
| 3 | 行番号付き表示（`-n`） | `-n` オプションで一致行に行番号を付けて表示する |
| 4 | 否定検索（`-v`） | `-v` オプションでパターンに一致しない行だけを表示する |
| 5 | 拡張正規表現（`-E`） | `-E` オプションで `5[0-9]{2}` のような正規表現を使い HTTP 5xx を抽出する |
| 6 | 件数カウント（`-c`） | `-c` オプションで一致した行数のみを数値で表示する |
| 7 | 前後の行を表示（`-C`） | `-C 1` で一致行の前後1行を合わせて表示する（コンテキスト確認） |
| 8 | 複数ファイルの検索（`-l` / `-H`） | `-l` で一致したファイル名のみ、`-H` でファイル名付きで表示する |
| 9 | 再帰検索（`-r`） | `-r` でディレクトリ配下の全ファイルを横断して検索する |
| 10 | 一致部分のみ抽出（`-o`） | `-o` と正規表現を組み合わせてログから `host=` の値だけを取り出す |
| 11 | パイプラインで絞り込み | `grep "ERROR" \| grep "\[db\]"` でコンポーネント別のエラーを抽出する |
| 12 | Slow response の検出 | キーワード検索でレスポンスタイムが遅い行を抽出する |

### ✏️ [sed.sh](sed.sh)

テキストの置換・削除・挿入をストリーム処理するコマンドです。設定ファイルの書き換えや環境切り替えで頻繁に使用します。  
サンプルデータ：設定ファイル (`/tmp/config.conf`)、アプリケーションログ (`/tmp/app.log`)

| No | タイトル | 概要 |
|---|---|---|
| 1 | 基本置換（`s/前/後/`） | `s/production/staging/` で最初の一致箇所を置換する |
| 2 | 全件置換（`g` フラグ） | `s/INFO/[INFO]/g` で行内の全マッチを置換する |
| 3 | 特定行の置換（行番号指定） | `2s/.../.../` で2行目だけを対象に置換する |
| 4 | パターン行の置換 | `DB_PASS=.*` にマッチする行のパスワード値を `****` でマスクする |
| 5 | 行の削除（`d`） | `/^#/d` でコメント行を、`/^$/d` で空行をそれぞれ削除する |
| 6 | 行番号での削除 | `1d` で1行目、`$d` で最終行を削除する |
| 7 | 特定行のみ表示（`-n` + `p`） | `-n` と `p` を組み合わせて `DB_` で始まる行だけを出力する |
| 8 | 行の挿入・追加（`i` / `a`） | `i` で指定行の前に、`a` で後に新しい行を挿入する |
| 9 | ファイル直接編集（`-i`） | `-i.bak` でバックアップを作成しながらファイルを直接書き換える |
| 10 | 複数操作の同時実行（`-e`） | `-e` を複数並べることで1コマンドで複数の置換を実行する |
| 11 | 後方参照（`\1`） | グループキャプチャを使い `KEY=VALUE` を `KEY: VALUE` 形式に変換する |
| 12 | タイムスタンプの除去 | ログ行頭の日時パターンを正規表現で削除してメッセージ部分だけ残す |
| 13 | 環境切り替え（パイプライン） | `sed` を複数パイプでつないで本番設定をステージング設定に一括変換する |

### 📊 [awk.sh](awk.sh)

フィールド単位の処理・集計・レポート生成が得意なコマンドです。ログの定量分析やメトリクス集計に使用します。  
サンプルデータ：アプリケーションログ (`/tmp/app.log`)、ノードメトリクス TSV (`/tmp/metrics.tsv`)

| No | タイトル | 概要 |
|---|---|---|
| 1 | フィールド抽出（`$N`） | `$1 $2 $3 $6` で日時・ログレベル・パスだけを取り出す |
| 2 | 条件フィルタ | `$3 == "ERROR"` でフィールドの値が一致する行だけを表示する |
| 3 | 数値比較 | `$8 > 500` でレスポンスタイムが500ms超の行を抽出する |
| 4 | 集計（連想配列） | `count[$3]++` でログレベルごとの件数を集計する |
| 5 | ステータスコード集計 | HTTPステータスコードをキーに件数を集計してソートする |
| 6 | 平均値の計算 | `sum += $8; cnt++` でレスポンスタイムの平均を `printf` で出力する |
| 7 | 最大・最小 | 比較しながら最大値・最小値とその行全体を保持して最後に出力する |
| 8 | フィールド区切り変更（`-F`） | `-F'\t'` でTSVファイルを処理し、CPU使用率80%超のノードを抽出する |
| 9 | NR / NF の利用 | `NR > 2` でヘッダー行をスキップ、`END` ブロックで最終行を表示する |
| 10 | BEGIN / END ブロック | `BEGIN` でヘッダーを出力し、`END` で集計サマリーを表示するレポートを作成する |
| 11 | ノード別メトリクスサマリー | `SUBSEP` を使った複合キーでノードごとの CPU / MEM / DISK を一覧表示する |
| 12 | エラー率の計算 | 総行数とエラー行数からエラー率を `printf` でパーセント表示する |

### 🔧 [jq.sh](jq.sh)

JSON データの抽出・変換・集計を行うコマンドです。Kubernetes や各種APIのレスポンスを処理する際に使用します。  
サンプルデータ：Kubernetes Pod 一覧 (`/tmp/pods.json`)、アラート一覧 (`/tmp/alerts.json`)

| No | タイトル | 概要 |
|---|---|---|
| 1 | フィールド取得 | `.items[].metadata.name` でネストしたフィールドの値を一覧表示する |
| 2 | 複数フィールドの取得 | `{name: .metadata.name, phase: .status.phase}` で必要なキーだけを持つオブジェクトに整形する |
| 3 | 条件フィルタ（`select`） | `select(.status.phase == "Running")` で条件に一致する要素だけ抽出する |
| 4 | 配列変換（`map`） | `[.items[] \| select(...)]` で条件フィルタした結果を配列として取得する |
| 5 | 文字列補間と raw 出力（`-r`） | `"\(.name): \(.phase)"` で複数フィールドを結合し、`-r` でクォートなしで出力する |
| 6 | 件数取得（`length`） | `.items \| length` でPodの総数を、`select` と組み合わせてRunning数を取得する |
| 7 | グループ化・集計（`group_by`） | `group_by(.phase)` でステータス別にグループ化し件数を集計する |
| 8 | ソート（`sort_by`） | `sort_by(-.value)` でアラートを値の降順に並べ替えて表示する |
| 9 | 重複除去（`unique`） | `[.[].alertname] \| unique` でアラート種別の一覧を重複なしで取得する |
| 10 | 比較演算子によるフィルタ | `select(.severity == "critical")` や `select(.value > 80)` で条件絞り込みを行う |
| 11 | キー一覧取得（`keys`） | `.[0] \| keys` でオブジェクトのキー名を確認する |
| 12 | TSV 形式での出力（`@tsv`） | `@tsv` で複数フィールドをタブ区切りに変換し `column -t` で整形表示する（kubectl風） |
| 13 | 値のリストをカンマ区切りで出力 | `select` → `unique` → `join(", ")` でcriticalアラートのインスタンス一覧を1行で出力する |
| 14 | CSV 変換（`@csv`） | ヘッダー行と組み合わせて `@csv` でアラートデータをCSV形式に変換する |

### 🔎 [find.sh](find.sh)

ファイルシステムを横断してファイルを検索するコマンドです。  
古いログファイルの検出・削除や、ディスク逼迫時の大容量ファイル特定に使用します。  
サンプルデータ：`/tmp` および `/etc` 配下のファイル

| No | タイトル | 概要 |
|---|---|---|
| 1 | ファイル名で検索（`-name`） | `-name "*.log"` でファイル名のパターンに一致するファイルを検索する |
| 2 | ファイルタイプで絞り込み（`-type`） | `-type f`（ファイル）/ `-type d`（ディレクトリ）/ `-type l`（シンボリックリンク）で絞り込む |
| 3 | 更新時刻で検索（`-mtime`） | `-mtime -1` で24時間以内、`-mtime +7` で7日以上前に更新されたファイルを検索する |
| 4 | ファイルサイズで検索（`-size`） | `-size +100M` で100MB超、`-empty` で空ファイルを検索する |
| 5 | パーミッションで検索（`-perm`） | `-perm 777` や `/4000`（SUIDビット）でセキュリティ確認に使用する |
| 6 | 所有者・グループで検索 | `-user` / `-group` で特定ユーザー・グループが所有するファイルを検索する |
| 7 | 検索結果でコマンド実行（`-exec`） | `-exec ls -lh {} \;` で検索したファイルに対してコマンドを実行する |
| 8 | xargsと組み合わせ | `find ... \| xargs grep` で大量ファイルを効率よく横断検索する |
| 9 | 検索深さの制限（`-maxdepth`） | `-maxdepth 2` で検索階層を制限してパフォーマンスを向上させる |
| 10 | 複数条件の組み合わせ | `\( -name "*.log" -o -name "*.txt" \)` で複数条件をand/orで組み合わせる |
| 11 | 古いログファイルの確認 | `-mtime +30` で30日以上前のログを確認し `-delete` で削除する |
| 12 | 大容量ファイルの特定 | `-size +100M` とxargsを組み合わせてサイズ順にソートして表示する |
| 13 | 直近の設定変更確認 | `-mmin -60` で直近1時間に変更された設定ファイルを一覧表示する |
| 14 | world-writableファイル検出 | `-perm -o+w` で誰でも書き込めるファイルを検出してセキュリティ確認に使用する |
