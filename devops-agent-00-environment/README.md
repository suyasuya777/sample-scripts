# AWS DevOps Agent 初期設定手順

AWS マネージメントコンソールから AWS DevOps Agent を使用開始するまでの手順を説明します。

---

## 前提条件

- AWS マネージメントコンソールへのアクセス権限があること
- 対象リージョン: **東京（ap-northeast-1）**

---

## セットアップ手順

### Step 1 　AWS DevOps Agent を選択

AWS マネージメントコンソールにサインインし、サービス一覧から **AWS DevOps Agent** を選択します。

---

### Step 2 　リージョン確認 ＆ セットアップ開始

1. コンソール右上のリージョン表示が **東京（ap-northeast-1）** であることを確認
2. **「セットアップを開始」** をクリック

> ⚠️ リージョンが異なる場合は、東京リージョンに切り替えてから進めてください。

---

### Step 3 　エージェントスペースを作成

1. 新規作成するエージェントスペース名を入力
2. **「保存」** をクリック

保存が完了すると、以下のロールが **自動作成** されます。

| ロール | 説明 |
|---|---|
| エージェントスペースロール | エージェントが AWS リソースへアクセスするためのロール |
| ウェブアプリロール | ウェブアプリケーション用のアクセスロール |

---

### Step 4 　オペレーターアクセスを開く

作成されたエージェントスペースの画面で **「オペレーターアクセス」** をクリックします。

---

### Step 5 　インシデントレスポンスから調査を開始

**「インシデントレスポンス」** セクションを選択し、調査を始めます。

---

### Step 6 　調査を開始

**「調査を開始」** をクリックします。

---

### Step 7 　調査の詳細を入力

以下の項目を入力後、**「調査を開始」** をクリックします。

| 項目 | 説明 |
|---|---|
| 調査の詳細 | 調査の目的・背景など概要を記入 |
| 調査の出発点 | 調査を開始するきっかけとなった事象や情報を記入 |

---

## 手順フロー

```
コンソールにサインイン
        ↓
AWS DevOps Agent を選択
        ↓
リージョン（東京）を確認 → セットアップを開始
        ↓
エージェントスペース名を入力 → 保存
        ↓
オペレーターアクセスをクリック
        ↓
インシデントレスポンスを選択
        ↓
調査を開始 をクリック
        ↓
調査の詳細・調査の出発点を入力 → 調査を開始
```
